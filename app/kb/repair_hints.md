# Repair Hints for LocalScript Lua

## Назначение
Этот файл нужен для self-repair после первичной генерации модели.
Каждый hint должен быть коротким, понятным и пригодным для повторной генерации.

---

## Формат hint
Используй стиль:
- "Исправь путь к данным: ..."
- "Убери лишний текст вокруг кода."
- "Верни JSON с lua{...}lua вместо raw Lua."
- "Не используй JsonPath."
- "Добавь return."

---

## Категории ошибок

### 1. FORMAT_RAW_TEXT
**Симптом**
Ответ содержит объяснения, вводные слова или markdown fences.

**Hint**
Убери весь пояснительный текст и markdown. Верни только итоговый код.

### 2. FORMAT_JSON_WRAPPER_REQUIRED
**Симптом**
Нужен JSON со строками `lua{...}lua`, а модель вернула raw Lua.

**Hint**
Верни JSON-объект. Значения полей должны быть строками в формате `lua{...}lua`.

### 3. FORMAT_RAW_LUA_REQUIRED
**Симптом**
Нужен raw Lua, а модель вернула JSON.

**Hint**
Верни только Lua-код без JSON-обертки.

### 4. NO_RETURN
**Симптом**
В коде нет итогового `return`, хотя задача предполагает вычисление результата.

**Hint**
Добавь корректный итоговый `return`.

### 5. JSONPATH_USED
**Симптом**
Обнаружен JsonPath или псевдо-JsonPath.

**Hint**
Не используй JsonPath. Обращайся к данным только напрямую через `wf.vars` или `wf.initVariables`.

### 6. WRONG_DATA_PATH
**Симптом**
Использован неверный путь к данным.

**Hint**
Исправь путь к данным. Используй точное поле из контекста без выдуманных промежуточных узлов.

### 7. ARRAY_HELPER_REQUIRED
**Симптом**
В задаче требуется новый массив, а модель вернула обычную таблицу.

**Hint**
Для нового массива используй `_utils.array.new()`.

### 8. ARRAY_NORMALIZATION_BROKEN
**Симптом**
Неправильно обработан случай `object vs array`.

**Hint**
Проверь тип `table`, определи, являются ли ключи числовыми, и при необходимости оберни объект в `{t}`.

### 9. EMPTY_STRING_NIL_CHECK_MISSING
**Симптом**
В фильтрации проверяется только `nil` или только `""`.

**Hint**
В условиях фильтрации учитывай и `nil`, и пустую строку.

### 10. DATETIME_PARSE_BROKEN
**Симптом**
Неверно разобрана дата или время.

**Hint**
Используй явный разбор строки через `string.sub`, `string.match`, `tonumber` и `string.format`.

### 11. INVALID_JSON_OUTPUT
**Симптом**
Сломанный JSON, неэкранированные переводы строк или неверные кавычки.

**Hint**
Собери корректный JSON. Все Lua-фрагменты должны быть строками.

### 12. NON_DOMAIN_HELPERS
**Симптом**
Использованы выдуманные функции или внешние библиотеки.

**Hint**
Не используй внешние helper-функции. Разрешены только стандартный Lua и доменные `_utils.array.*`.

### 13. TOO_COMPLEX
**Симптом**
Решение сильно сложнее, чем нужно.

**Hint**
Упрости решение. Используй минимально достаточный код.

### 14. MUTATION_NOT_NEEDED
**Симптом**
Модель изменила входные данные, хотя требовалось просто вычислить значение.

**Hint**
Не изменяй исходные данные без необходимости. Верни только нужный результат.

### 15. SYNTAX_ERROR
**Симптом**
Ошибка Lua-синтаксиса: скобки, `end`, запятые, строки.

**Hint**
Исправь синтаксис Lua. Проверь `if`, `for`, `function`, скобки и `end`.

## Универсальный repair prompt
Используй этот шаблон во второй итерации:

Твоя предыдущая версия кода нарушает доменные правила.
Исправь код строго по замечаниям ниже.
Верни только итоговый код без пояснений.

Замечания:
- {repair_hint_1}
- {repair_hint_2}
- {repair_hint_3}

Исходная задача:
{task}

Предыдущий код:
{previous_code}

---

## Приоритет исправлений
Если ошибок несколько, исправляй в таком порядке:
1. формат ответа;
2. запрещенные паттерны;
3. путь к данным;
4. синтаксис;
5. доменная логика;
6. упрощение.

---

## Критерий выбора между двумя версиями
Предпочитай версию, которая:
1. проходит форматную проверку;
2. не использует JsonPath;
3. использует точный путь к данным;
4. синтаксически валидна;
5. короче и проще.
